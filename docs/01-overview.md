# 01. 概要と要件

## 1.1 コンセプト

**「君と育つモンスター」**

子供が学習に取り組むと、ゲーム内通貨「コイン」が貯まる。コインで餌や装備品を
買い、モンスターを育てていく。学習を続けるほどモンスターは強く、可愛く、賢く
育つ。

学習の「やらされ感」を、モンスターのお世話という能動的な動機に置き換えるのが
ねらい。

### コアループ

```
  学習する  ──→  コイン獲得  ──→  ショップで購入  ──→  モンスターのお世話
     ↑                                                          │
     └────────  モンスターから「もっと勉強しよう」  ←────────┘
```

このループを 5〜15 分のセッションで一回回せることを目標にする。子供の集中力を
踏まえ、一度の学習セッションは短く、達成感のある区切りを必ず提供する。

## 1.2 対象ユーザー

- **主ユーザー**: 幼児〜小学校高学年（おおよそ 4〜12 歳）
- **副ユーザー**: 保護者（学習履歴の確認、コンテンツの難易度設定）
- **デバイス**: スマホ / タブレット / PC（タッチ操作前提、マウス操作も可）

年齢幅が広いため、コンテンツ側で難易度を段階化し、UI は「字が読めない子でも
アイコンで操作できる」ことを基本方針にする。

## 1.3 学習コンテンツ（初期）

書き取り（kakitori ライブラリベース）を最初に提供する。

| コンテンツ | 対象 | 出題例 |
|---|---|---|
| 数字の書き取り | 4 歳〜 | 0〜9 の書き順 |
| ひらがなの書き取り | 5 歳〜 | あ〜ん（清音→濁音→拗音と段階化） |
| 漢字の書き取り | 6 歳〜 | 学年別配当漢字 |

将来は計算ドリル、読み上げ、英語のスペル、暗算などをプラグインとして追加できる
ようにする（→ [03-plugin-architecture.md](03-plugin-architecture.md)）。

## 1.4 機能要件

### MUST（MVP）

- 学習コンテンツを 1 つ以上選んで実施できる
- 学習結果に応じてコインが付与される
- モンスター 1 体を所有し、餌をあげたり装備をつけたりできる
- ショップで餌・装備を購入できる
- データはローカル（IndexedDB）に保存され、ブラウザを閉じても残る
- オフラインでも全機能が動作する（PWA）

### SHOULD（v1）

- モンスターのレベルアップと進化
- 学習履歴の可視化（日別グラフ、保護者画面）
- 複数の学習プラグインを切り替えられる
- 称号・実績・コレクション要素

### COULD（将来）

- 複数モンスター飼育
- フレンドとのコイン交換 / 対戦
- クラウド同期（端末間バックアップ）
- 自作プラグイン投稿

### WON'T（当面やらない）

- 課金要素（子供向けである以上、慎重に扱う）
- 広告
- チャット・SNS 連携
- リアルタイム通信を要する対戦

## 1.5 非機能要件

| 項目 | 要件 |
|---|---|
| 動作環境 | モダンブラウザ（Chrome / Safari / Edge の最新2バージョン） |
| 起動時間 | 初回 3 秒以内、2 回目以降 1 秒以内（Service Worker キャッシュ） |
| オフライン | 全機能オフラインで動作。ネットワーク必須機能を作らない |
| データ保存 | IndexedDB。失敗時はユーザに明示し、再試行できる |
| プライバシー | 個人情報を一切収集しない。学習履歴はローカルに留める |
| アクセシビリティ | 読み仮名、音声読み上げ、配色のコントラスト確保 |
| 多言語 | 当面は日本語のみ。i18n の口は開けておく |

## 1.6 スコープ外の前提

- アカウント作成・サインインは作らない。端末ローカルで完結。
- マネタイズは設計の範囲外。実装着手時にも追加しない。
- 親モードのパスワード / 端末ロックは v1 範囲外（ただし、ショップで間違って
  全部買わせない仕組みは MVP に含める。詳細は ui-ux.md）。

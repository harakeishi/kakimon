# 05. UI/UX 設計

## 5.1 デザイン原則

1. **字が読めない子でも遊べる**。重要な操作はアイコン + ラベル。
2. **ボタンは大きく**。タップターゲットは最小 56px、推奨 72px 以上。
3. **モンスターを画面の主役にする**。アプリを開いたら必ずモンスターが見える。
4. **やったことに対する反応を返す**。タップ、給餌、装備変更には必ず音 or
   アニメーション。
5. **失敗を責めない**。間違えても優しいフィードバック。書き取りで間違えたら
   「もう一回」を促すだけ。

## 5.2 画面一覧

| 画面 | 役割 | 主な遷移先 |
|---|---|---|
| ホーム | モンスター表示・お世話・主要メニュー | 学習選択 / ショップ / 装備 / 図鑑 |
| 学習選択 | プラグイン（学習コンテンツ）一覧 | 難易度選択 |
| 難易度選択 | 難易度 + 問題数の設定 | 学習プレイ |
| 学習プレイ | プラグインがマウントされる | 学習結果 |
| 学習結果 | 獲得コイン・経験値の演出 | ホーム |
| ショップ | 餌・装備の購入 | ホーム |
| 装備管理 | 所有装備の付け外し | ホーム |
| 図鑑 | 出会ったモンスター・コレクション | ホーム |
| 保護者画面 | 学習履歴グラフ・難易度推奨 | (PIN 入力後) |
| 設定 | 音量・テーマ・データ消去 | (保護者画面の中) |

## 5.3 主要画面のワイヤー

文章でレイアウトの意図だけ示す。実装時に Figma で詰める。

### ホーム画面

```
┌──────────────────────────────────────┐
│  [♡ HP][😋 Hunger][😊 Mood][💰 999]  │ ← 上部ステータスバー
├──────────────────────────────────────┤
│                                      │
│           [モンスターの絵]            │
│           (息遣いアニメ)              │
│           「タップで撫でる」          │
│                                      │
├──────────────────────────────────────┤
│  [📚 勉強する][🍎 ごはん][🛍 ショップ]│ ← 主要アクション (大きい)
├──────────────────────────────────────┤
│  [🎽 装備][📖 図鑑][⚙ 設定]          │ ← サブメニュー
└──────────────────────────────────────┘
```

- モンスターをタップすると喜ぶ（mood +）。
- 主要アクション 3 つは横並びの大ボタン。
- ステータスバーは数字も出すが、アイコンに色（緑→黄→赤）で状態を表現する。

### 学習選択画面

- プラグインがカードとしてグリッド表示される。
- カードには `manifest.icon`, `name`, `ageHint`, `category` を表示。
- カテゴリで絞り込みできる（writing / reading / math）。
- 「最近やったコンテンツ」を上に出す。

### 学習プレイ画面

- 上部に「のこり 3 / 10」と「やめる」ボタン。
- 中央に target 要素（プラグインが自由に描画）。
- 「やめる」は確認ダイアログを挟む（誤タップ防止）。
- ヘルプ（書き順の見本など）はプラグイン側責務。

### 学習結果画面

- でかでかとコイン獲得演出。「やった〜！」のセリフ。
- スコアに応じてモンスターが喜ぶアニメ。
- 「もう一回」「ホームに戻る」の 2 択。「もう一回」は同じ難易度・同じプラグイン
  でもう一度。

### ショップ画面

- 上部にコイン残高。
- タブで「ごはん」「装備」を切り替え。
- カード内に price と「買う」ボタン。
- 一度に大量買いを防ぐため、確認ダイアログを挟む（特に 10 コイン以上の
  アイテム）。

## 5.4 保護者向け配慮

### 保護者画面

- ホームに「保護者向け」ボタンは出さない。設定アイコン → 長押し or
  数字パズル（小学校低学年でも解けない年齢ゲート）で入る。
- 学習履歴の表示：
  - 日別の学習時間グラフ
  - プラグイン別の正答率
  - 直近 30 日分
- データ消去はここから。

### 安全機構

- 課金一切なし、広告なし。外部リンクなし。
- アプリ「外」に出る経路を最小化する。

## 5.5 アクセシビリティ

- **読み仮名**: 漢字には基本的にふりがな。OFF も可能（保護者画面）。
- **音声読み上げ**: ボタンや結果メッセージは音声で再生する（v1）。
- **配色**: WCAG AA を目安。色だけで状態を伝えない（アイコン or 文字を併用）。
- **モーションリダクション**: OS 設定（`prefers-reduced-motion`）を尊重し、
  激しいアニメは抑制する。
- **片手操作**: スマホで親指の届く位置に主要ボタンを配置する。

## 5.6 アセット

- モンスターのドット絵 or ベクター。実装着手時にデザインアセットを決める。
- 暫定的にはオリジナルの低解像度ドット絵で開発する。
- BGM は短いループ、SE は短く弾ける音。子供が長時間聞いても疲れない音作り。

## 5.7 マイクロインタラクション

| イベント | 反応 |
|---|---|
| アプリ起動 | モンスターが伸びをする |
| 撫でる | きゅるん、ハートが浮く |
| 給餌 | もぐもぐ、満腹度が回復 |
| 装備変更 | きらりエフェクト、新しい見た目が即反映 |
| 学習開始 | モンスターが「がんばれ〜」のセリフ |
| 学習成功 | コイン獲得演出、レベルアップなら盛大に |
| 学習中断 | モンスターが寂しそうな顔（責めない範囲で） |

## 5.8 多言語

UI 文字列は `i18n` キーで管理する設計にしておく。実装は ja のみ。
キーは `home.title`, `study.startButton` のようなドット記法で統一。

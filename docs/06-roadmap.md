# 06. ロードマップ

設計フェーズ完了後の実装ロードマップ。各フェーズの完了基準を明示する。

## Phase 0: プロジェクト基盤（着手前）

### 目的

開発を始められる状態にする。

### 作業

- [ ] pnpm workspace 初期化
- [ ] `apps/web` の Vite + React + TS テンプレート
- [ ] `packages/plugin-api` の雛形（型のみ）
- [ ] ESLint / Prettier / Vitest 設定
- [ ] vite-plugin-pwa の最低限の設定（manifest, service worker）
- [ ] CI（GitHub Actions: typecheck + test + build）
- [ ] 静的ホスティングへのデプロイパイプライン

### 完了基準

- `pnpm install && pnpm dev` で空のホーム画面が立ち上がる。
- `pnpm test` がグリーン。
- PR を作ると CI が走る。

## Phase 1: コア基盤 + 1 プラグイン + 最小モンスター

### 目的

コアループ（学習→コイン→給餌→お世話）を一周回せる状態にする。

### 作業

- [ ] Domain 層（Monster, Wallet, Inventory, StudySession, RewardCalculator）
- [ ] Dexie スキーマと Repository
- [ ] Plugin Host（registry, ライフサイクル）
- [ ] `@kakimon/plugin-writing-hiragana`（清音のみ、kakitori 統合）
- [ ] ホーム画面（モンスター表示・お世話ボタン）
- [ ] 学習選択 → プレイ → 結果の一連
- [ ] ショップ画面（餌のみ 3 種類）
- [ ] 設定画面（音量・データ消去）

### 完了基準

- ひらがなを 5 文字書いたら 5 コイン以上もらえる。
- もらったコインで餌を買い、モンスターに食べさせると Hunger が回復する。
- ブラウザを閉じて開き直しても状態が保たれている。
- オフラインでも上記が全部できる。

## Phase 2: コンテンツとお世話の拡張

### 目的

学習の幅を増やし、育成の手応えを作る。

### 作業

- [ ] `@kakimon/plugin-writing-numbers`
- [ ] `@kakimon/plugin-writing-kanji`（小1配当）
- [ ] 装備 5 種類とショップでの購入・装備管理画面
- [ ] モンスターのステージ進化（egg → baby → child）
- [ ] レベル別ステータス成長
- [ ] **時間経過の tick 処理（hunger / cleanliness / mood / hp の減衰）**
- [ ] **lifeState 遷移と健康度に応じた UI（weak / sick / dying の演出と警告）**
- [ ] **死亡 → お墓 → 新しいタマゴの再スタートフロー**（コイン・履歴は持ち越し）
- [ ] **保護者向け「やさしいモード（死亡なし）」設定**
- [ ] BGM / SE
- [ ] アニメーション（撫でる・喜ぶ・もぐもぐ）

### 完了基準

- 3 つの書き取りプラグインが同じ UI 上で切り替えて遊べる。
- 装備によってモンスターの見た目が変わる。
- 「baby → child」の進化演出が動く。
- 端末の時計を 1 週間進めるとモンスターが死亡し、新しいタマゴから再開できる。
  コインと装備は引き継がれる。
- 「やさしいモード」を ON にすると同じ操作で死亡せず体力 1 で踏みとどまる。

## Phase 3: 保護者画面とアクセシビリティ

### 目的

「親が安心して子供に渡せる」状態にする。

### 作業

- [ ] 保護者画面（学習履歴グラフ、ふりがな on/off）
- [ ] 年齢ゲート（保護者画面への入り口）
- [ ] 音声読み上げ
- [ ] `prefers-reduced-motion` 対応
- [ ] バックアップ / インポート（JSON）

### 完了基準

- 保護者が「今週どれくらい勉強したか」を見られる。
- データを別端末へ JSON 経由で移行できる。

## Phase 4: コンテンツ拡充

### 目的

長く遊べるコンテンツ量を作る。

### 作業

- [ ] 漢字を小2〜小6まで
- [ ] 計算プラグイン（たし算・ひき算）
- [ ] 読み上げプラグイン（音声合成 or 録音）
- [ ] 図鑑・コレクション要素
- [ ] 実績（連続学習日数など）

### 完了基準

- 1 日 5〜10 分のプレイを 3 ヶ月続けても飽きない程度のコンテンツ量。

## やらないことの再確認

- 課金、広告、SNS、アカウント機能、サーバ連携
- 競争要素（ランキング等）
- 「家出」「破壊」など、死亡以外の喪失バリエーション

これらは「子供向けの学習アプリ」というプロダクトの軸を守るために、ロードマップ
から意図的に外してある。要求が来たときは外している理由に立ち返って判断する。

なお、**ペットの死亡は仕様として採用**している（→ `04-domain-model.md` 4.4 節）。
長期間世話をしないとモンスターは段階的に弱り、最終的に死亡する。
ただし致死までの猶予は 1 週間以上、学習成果（コイン・履歴）は持ち越し、
保護者向けに「やさしいモード（死亡なし）」を用意することで、子供向けに
配慮した設計にしている。
